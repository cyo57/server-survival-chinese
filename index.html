流量类型

STATIC（GET）

图片、CSS、JS → 存储。90% 缓存命中率。

READ（GET）

API 数据读取 → SQL 数据库。40% 缓存命中率。

WRITE（POST/PUT）

数据库写入 → SQL 数据库。不缓存。

UPLOAD（POST）

文件上传 → 存储。不缓存，处理耗时。

SEARCH（GET）

复杂查询 → SQL 数据库。15% 缓存命中，处理耗时。

机制

升级

点击计算/数据库/缓存可升级容量（T1-T3）。

负载均衡器使用轮询分发流量。

失败条件

声望降至 0% 或资金 ≤ -$1000。

高级服务

消息队列

最多缓冲 200 请求。放在防火墙后可平滑 DDoS，高峰前置于计算避免过载，是廉价的抗峰值方案。

推荐流向：防火墙 → 队列 → 负载均衡 或 负载均衡 → 队列 → 计算

内存缓存

35-65% 命中率的缓存层。命中后无需访问数据库/存储，可升级提升命中率，显著降低数据库成本。

推荐流向：计算 → 缓存 → 数据库/存储
